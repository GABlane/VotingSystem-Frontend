# Vercel Deployment Checklist

Quick checklist to deploy frontend to Vercel.

## Pre-Deployment

- [ ] Frontend working locally (`npm run dev`)
- [ ] Build succeeds (`npm run build`)
- [ ] `.env.local` has all variables
- [ ] Backend deployed on Railway
- [ ] Backend URL copied from Railway

## Push to GitHub

- [ ] Code committed to Git
- [ ] Pushed to GitHub repository
- [ ] Repository is accessible

## Vercel Setup

- [ ] Vercel account created
- [ ] Project imported from GitHub
- [ ] Root directory set (if monorepo: `frontend`)

## Environment Variables Set

Add these in Vercel dashboard:

- [ ] `NEXT_PUBLIC_API_URL` = `https://web-production-90542.up.railway.app`
- [ ] `NEXT_PUBLIC_SUPABASE_URL` = (your Supabase URL)
- [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY` = (your Supabase anon key)

## Deployment

- [ ] First deployment triggered
- [ ] Build logs show success
- [ ] Deployment complete
- [ ] Vercel URL generated

## Post-Deployment Testing

Test your Vercel URL:

- [ ] Landing page loads
- [ ] Can navigate to login
- [ ] Admin login works
- [ ] Dashboard loads
- [ ] Projects display
- [ ] Can create project
- [ ] QR code displays
- [ ] Public voting page works

## Update Railway Backend

**CRITICAL STEP:**

- [ ] Copy Vercel URL (e.g., `https://your-app.vercel.app`)
- [ ] Go to Railway dashboard
- [ ] Update `FRONTEND_URL` variable to Vercel URL
- [ ] Redeploy Railway backend
- [ ] Test CORS (API calls from frontend should work)

## Verify End-to-End

- [ ] Create a test project from admin dashboard
- [ ] View QR code
- [ ] Visit public voting page
- [ ] Cast a vote
- [ ] Verify vote count updates in real-time
- [ ] Check admin results page shows vote

## Optional

- [ ] Custom domain configured
- [ ] Analytics enabled
- [ ] Auto-deploy enabled
